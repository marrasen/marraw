Recommended direction · consolidated spec · v1

Cull → Focus on a Cinema canvas

One zero-chrome, edge-to-edge canvas serves both jobs. CULL triages a shoot fast against a full-size confirm loupe; DEVELOP perfects one frame with the complete tool stack a keystroke away. Chrome is glass and auto-hides, so the photograph always dominates. Ships dark-first with a full light theme.

01 · CANVAS
Zero-chrome, image-maximal

The photo fills the window. Every deck is floating glass that auto-hides when idle; the image is draggable behind pinned panels — nothing pushes it.

02 · MODES
Two modes, one shared HUD

A top segmented control (Library · Cull · Develop · Export) + ⌘K stay constant everywhere, so navigation and muscle memory never move between tasks.

03 · CONTROLS
Three-tier control model

Quick-dock for the ~6 dials you ride constantly · pinnable Develop drawer for the full stack · ⌘K + hotkeys to reach anything without opening a panel.

LIBRARY

The home base — browse, filter, and triage the whole shoot

The one structured surface: a folder rail, a virtualized thumbnail grid sized by a live slider, and the persistent Develop panel tracking the focused frame. Filters and counts sit inline; Enter drops the focused photo into the Cull loupe. This is where every session starts.

ORGANIZE

Right-click to rename, reorder, locate, and remove

Every row in the rail carries a context menu. Rename sets a display alias only — it never touches the folder on disk (a separate "Rename on disk…" is offered where it does). Locate on disk opens the folder in Explorer. Remove only drops it from the library; the files always stay put. Reorder is drag, or the arrow items here.

Light theme · same system, inverted

The full light theme

Every surface has a light twin. Below: the structured Library, a cinema Cull, and the dense Develop drawer. The remaining screens follow these same rules.
